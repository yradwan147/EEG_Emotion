\section{Experimental Details}
\label{sec:supp_exp}

\noindent\textbf{Hardware.} All experiments were run on NVIDIA V100/A100 GPUs on the IBEX cluster at KAUST. Training time per experiment: $\sim$50 minutes (50 epochs $\times$ 1 min/epoch).

\noindent\textbf{Hyperparameters.} AdamW optimizer with multi-LR (backbone: $10^{-4}$, heads: $5 \times 10^{-4}$), cosine annealing to $10^{-6}$, weight decay $0.05$, gradient clipping $1.0$, batch size 64, dropout $0.1$, label smoothing $0.1$.

\noindent\textbf{Emotion vector extraction.} For each emotion, 30 stories are generated using diverse topic prompts. Hidden states are extracted from the 67\textsuperscript{th}-percentile layer, averaged across tokens (excluding the first 50 prompt tokens) and stories, then the global mean is subtracted.

\noindent\textbf{Total compute.} Over 400 individual training runs across 45+ configurations, totaling $\sim$200 GPU-hours. All results use 5-seed evaluation (seeds: 3407, 42, 123, 456, 789).

\section{Multi-LLM Emotion Vector Analysis}
\label{sec:supp_llm}

We extract emotion vectors from 8 models across 6 families to test whether LLM emotion geometry is universal or specific to a single model family. For each model, we compute the first principal component of the 9 emotion vectors and correlate it with the ground-truth valence ordering. \cref{tab:supp_llm} summarizes the results.

Instruction-tuned models (Qwen, Mistral) exhibit significant valence structure ($p < 0.05$), while base models (Pythia, Bloom) show no structure. Cross-family RDM agreement between instruction-tuned models is high: Qwen--Mistral $r = 0.93$, Qwen--Phi-2 $r = 0.52$, Mistral--TinyLlama $r = 0.63$ (all $p < 0.01$). Notably, smaller instruction-tuned models (Qwen2.5-1.5B) show stronger emotion geometry than larger ones (Qwen2.5-7B), suggesting that emotion representations become more distributed and harder to extract as model capacity increases. In practice, we use the smallest available instruction-tuned model (Qwen2.5-0.5B), as it provides equivalent downstream performance (cross-model agreement $r > 0.89$ within the Qwen family).

\section{Additional Architecture Exploration}
\label{sec:supp_explore}

We explored several additional architecture directions beyond the main results:

\begin{itemize}
    \item \textbf{Cross-attention readout.} We tested Perceiver-style~\cite{jaegle2021perceiver} learnable query tokens attending over backbone features. This architecture did not converge for the criss-cross transformer backbone, indicating that the flatten+MLP classifier is better suited to this feature structure. This finding informs future EEG architecture design.
    \item \textbf{Masked autoencoder pretraining.} We investigated VideoMAE-style~\cite{tong2022videomae} pretraining on CBraMod. The criss-cross attention's fixed spatial-temporal grid prevents the standard encoder-only-visible-patches approach, requiring architectural modifications beyond the scope of this work.
    \item \textbf{Hyperparameter interaction.} Combining the two best individual hyperparameters ($p_{\text{aug}} = 0.6$ and $\tau_t = 0.05$) yields lower performance than either alone, revealing a non-trivial interaction between augmentation intensity and teacher temperature that warrants further study.
    \item \textbf{Training duration.} Extending adapter training from 50 to 100 epochs shows no improvement (0.629 vs.\ 0.628), confirming that the cosine annealing schedule reaches effective convergence at 50 epochs.
\end{itemize}
